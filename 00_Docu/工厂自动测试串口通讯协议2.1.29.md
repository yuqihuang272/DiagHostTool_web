# CVTE Factory Auto Test Serial Communication Protocol

**Version**: 2.1.51
**Author**: CVTE
**Last Updated**: 2026-02-27

## Overview

This document describes the serial communication protocol for CVTE factory auto testing. It is used for TV board testing and configuration.

## Serial Port Settings

| Parameter | Value |
|-----------|-------|
| Data Bits | 8 |
| Parity | None |
| Stop Bits | 1 |
| Baud Rate | 115200 |
| Data Interval | Same packet data interval < 100ms |

## Packet Format

```
Byte Index | Value       | Description
-----------|-------------|---------------------------
0x00       | 0xFF        | Sync byte
0x01       | 0x33        | Start byte
0x02       | 0xNN (>=6)  | Packet length
0x03...    | 0x00-0xFF   | Data (Protocol ID + Parameters)
0xNN-1     | 0x00-0xFF   | Checksum
```

### Checksum Calculation

```
Checksum = 0x100 - (data[2] + data[3] + ... + data[NN-2])
```

All data uses unsigned bytes. The checksum is calculated from byte index 2 to the byte before the checksum.

## Protocol Types

| Protocol ID | Name | Description |
|-------------|------|-------------|
| 0x01 | RF Auto Test | RF testing |
| 0x02 | Auto White Balance | White balance adjustment |
| 0x03 | Factory Auto Test | Factory testing commands |
| 0x04 | Auto Test Signal Generator | Signal generation |

## ACK Response

All commands may receive an ACK response:

```
Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x0C  | Packet length (12)
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x01  | ACK command ID
0x05       | 0x00-0x02 | Error code
0x06       | 0x00-0xFF | Command ID being acknowledged
0x07-0x0A  | 0x00  | Pocket index (4 bytes)
0x0B       | 0xXX  | Checksum
```

### Error Codes

| Value | Description |
|-------|-------------|
| 0x00 | OK |
| 0x01 | Unknown command ID |
| 0x02 | Parameter error |

## Command Details

### 4.2 Get TV Checksum (0x12/0x13)

**Request (PC→TV):**
```
Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x06  | Packet length
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x12  | GET_CHECKSUM command
0x05       | 0xE5  | Checksum
```

**Response (TV→PC):**
```
Byte Index | Value    | Description
-----------|----------|---------------------------
0x00       | 0xFF     | Sync byte
0x01       | 0x33     | Start byte
0x02       | 0xNN     | Packet length
0x03       | 0x03     | Factory auto test protocol
0x04       | 0x13     | RET_CHECKSUM command
0x05..NN-2 | 0x00-0xFF | Checksum data (ASCII)
0xNN-1     | 0xXX     | Checksum
```

Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x06  | Packet length
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x14  | GET_SOURCE command
0x05       | 0xE3  | Checksum
```

**Response (TV→PC):**
```
Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x07  | Packet length
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x15  | RET_SOURCE command
0x05       | 0xXX  | Source ID (see enumeration)
0x06       | 0xXX  | Checksum
```

### 4.4 Set Source (0x16)

**Request (PC→TV):**
```
Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x07  | Packet length
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x16  | SET_SOURCE command
0x05       | 0xXX  | Source ID to set
0x06       | 0xXX  | Checksum
```

**Checksum Calculation:** `checksum = 0x100 - (0x07 + 0x03 + 0x16 + source_id)`

**Examples:**
| Source | Source ID | Complete Packet |
|--------|-----------|-----------------|
| ATV | 0x00 | `FF 33 07 03 16 00 E0` |
| DTV | 0x01 | `FF 33 07 03 16 01 DF` |
| HDMI1 | 0x08 | `FF 33 07 03 16 08 D8` |
| HDMI2 | 0x09 | `FF 33 07 03 16 09 D7` |
| HDMI3 | 0x0A | `FF 33 07 03 16 0A D6` |
| VGA | 0x06 | `FF 33 07 03 16 06 DA` |
| AV1 | 0x0F | `FF 33 07 03 16 0F D1` |

**Response (TV→PC):** ACK

### 4.13 Get MAC Address (0x0C/0x0D)

**Request (PC→TV):**
```
Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x06  | Packet length
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x0C  | GET_MAC_ADDR command
0x05       | 0xEB  | Checksum (corrected from 0xEC)
```

**Response (TV→PC):**
```
Byte Index | Value | Description
-----------|-------|---------------------------
0x00       | 0xFF  | Sync byte
0x01       | 0x33  | Start byte
0x02       | 0x0C  | Packet length (12)
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x0D  | RET_MAC_ADDR command
0x05       | D0    | MAC byte 0
0x06       | D1    | MAC byte 1
0x07       | D2    | MAC byte 2
0x08       | D3    | MAC byte 3
0x09       | D4    | MAC byte 4
0x0A       | D5    | MAC byte 5
0x0B       | 0xXX  | Checksum
```

MAC address format: `D0:D1:D2:D3:D4:D5`

**Request (PC→TV):**

0x01       | 0x33  | Start byte
0x02       | 0x06  | Packet length
0x03       | 0x03  | Factory auto test protocol
0x04       | 0x31  | GET_IP_INFO command
0x05       | 0xC6  | Checksum
```

**Response (TV→PC):**
```
Byte Index | Value    | Description
-----------|----------|---------------------------
0x00       | 0xFF     | Sync byte
0x01       | 0x33     | Start byte
0x02       | 0xNN     | Packet length
0x03       | 0x03     | Factory auto test protocol
0x04       | 0x32     | RET_IP_INFO command
0x05..NN-2 | ASCII    | IP address string
0xNN-1     | 0xXX     | Checksum
```

IP address is returned as ASCII string (e.g., "192.168.1.100")
